# VibeOS Project Status

Use this skill for the strategic, founder-style view.

## Workflow

1. Read:
   - `project-definition.json`
   - `docs/product/PRODUCT-ANCHOR.md`
   - `docs/ENGINEERING-PRINCIPLES.md`
   - `docs/planning/DEVELOPMENT-PLAN.md`
   - `docs/planning/WO-INDEX.md`
   - `.vibeos/findings-registry.json`
   - `.vibeos/audit-reports/`
   - `.vibeos/session-audits/`
   - `docs/research/RESEARCH-REGISTRY.md`
   - `docs/decisions/DEVIATIONS.md`
2. Translate technical progress into product, delivery, and risk meaning.
3. Report:
   - bottom line
   - program stage
   - confidence
   - evidenced progress
   - what still needs to happen
   - major risks
   - strategic priorities
   - decisions needed from the user

## Rules

- This is not a backlog dump.
- If confidence is mixed, say so plainly.
- Use business meaning first, IDs second.
