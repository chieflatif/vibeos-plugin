# WO-012: Project Intake Integration

## Status

`Draft`

## Phase

Phase 2: Product Discovery & Planning

## Objective

Ensure project intake answers feed correctly into the decision engine, with pre-fill from project-definition.json and validation of required fields.

## Scope

### In Scope
- [ ] Map each intake question to decision engine input fields
- [ ] Pre-fill intake answers from project-definition.json when fields match
- [ ] Skip questions that are fully answered by pre-fill (with confirmation)
- [ ] Validate all required fields have answers before decision engine runs
- [ ] Handle missing/incomplete project-definition.json gracefully
- [ ] Error messages in plain English when validation fails

### Out of Scope
- Intake questionnaire design (already in PROJECT-INTAKE.md)
- Decision engine logic (already in decision-engine/)
- Plan generation (WO-009)

## Dependencies

| Dependency | Type | Status |
|---|---|---|
| WO-009 | Must complete first | Draft |
| project-definition.json | Generated by WO-008 | Draft |

## Impact Analysis

- **Files modified:** skills/plan.md (intake integration logic)
- **Systems affected:** Data flow between discover and plan skills

## Acceptance Criteria

- [ ] AC-1: Every intake question maps to at least one decision engine input
- [ ] AC-2: Pre-fill correctly reads from project-definition.json
- [ ] AC-3: Pre-filled answers shown to user for confirmation (not silently assumed)
- [ ] AC-4: Missing required fields produce clear error with which field and why
- [ ] AC-5: Works when project-definition.json is absent (all questions asked manually)
- [ ] AC-6: Works when project-definition.json is partial (only missing fields asked)
- [ ] AC-7: Decision engine receives complete, validated input

## Test Strategy

- **Unit:** Test pre-fill logic with full, partial, and missing project-definition.json
- **Integration:** Run plan skill with pre-existing project-definition.json, verify fewer questions asked
- **Validation:** Attempt to proceed with missing required fields, verify rejection

## Implementation Plan

### Step 1: Create Field Mapping
- Map each of 18 intake questions to project-definition.json fields
- Map intake answers to decision engine input keys
- Document which fields are required vs. optional

### Step 2: Implement Pre-fill Logic
- Read project-definition.json if exists
- Match fields to intake questions
- Present pre-filled answers for user confirmation
- Ask remaining questions normally

### Step 3: Implement Validation
- Check all required fields populated after intake
- Clear error messages for missing fields
- Allow user to provide missing answers interactively

### Step 4: Wire to Decision Engine
- Transform validated intake answers to decision engine input format
- Pass to decision engine trees
- Handle decision engine errors gracefully
